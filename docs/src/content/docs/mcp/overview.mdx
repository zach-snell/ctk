ctk implements the [Model Context Protocol](https://modelcontextprotocol.io/)
over stdio. When you run `ctk mcp`, it starts a server that AI agents can call
to interact with Confluence Cloud.

## Starting the Server

```bash
ctk mcp
```

The server reads credentials from environment variables or stored credentials
(same as the CLI).

## Write Gating

Write operations are **disabled by default**. When writes are disabled, tools
like `manage_pages` only expose read actions (`get`, `list`, `get_children`,
etc.). Create, update, delete, and move actions return an error.

To enable writes:

```bash
CTK_ENABLE_WRITES=true ctk mcp
```

The tool descriptions dynamically reflect which actions are available based on
the `CTK_ENABLE_WRITES` setting.

## Disabling Tools

You can disable specific tools with the `CTK_DISABLED_TOOLS` environment
variable:

```bash
CTK_DISABLED_TOOLS="manage_comments,manage_attachments" ctk mcp
```

<Aside type="note">
All responses are flattened and stripped of Confluence API bloat (`_links`,
`_expandable`, `extensions`, etc.). Page bodies are converted from XHTML
storage format to markdown. Responses larger than 40KB are truncated and the
full output is saved to `/tmp/ctk-logs/`.
</Aside>
